JavaScript中声明的提升

发表于 2017-01-04 | 分类于 Front-end |

前言

本文主要记录和总结了JavaScript中关于声明提升的概念和相关注意事项。

提升（Hoisting）

我们常习惯将var a = 2; 看作是一个声明，而实际上JS引擎并不这么认为。它将var a和a = 2当作两个单独的声明，第一个是编辑阶段的任务，而第二个是执行阶段的任务。

这意味着无论作用域中的声明出现在什么地方，都将在代码本身被执行前首先进行处理，可以将这个过程形象地想象成所有的声明（函数和变量）都会被“移动”到各自作用域的最顶端，这个过程称为提升(Hoisting)。

注意

  • 包括变量和函数在内的所有声明都会在任何代码被执行前首先被处理。
  • 只有声明本身会被提升，而赋值或其他运行逻辑会留在原地。
  • 变量声明和函数声明会被提升，但是函数表达式声明不会被提升。
  • 变量声明和函数声明都会被提升，但是当两者同时出现时，首先提升的是函数声明，其次才是变量声明。
